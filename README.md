# FinGuard AI

A Java-based framework for evaluating AI agent responses in financial contexts. It checks that AI-generated answers (e.g., for loan products) stay **grounded** in a configurable knowledge base—detecting hallucinations such as incorrect loan amounts or APR ranges—and supports testing via a mock loan agent (SUT).

**Tech stack:** Java 21, Maven, LangChain4j, TestNG, Jackson, SLF4J.

## Folder and file descriptions

### Root

| Path | Purpose |
|------|--------|
| `pom.xml` | Maven POM: defines `com.rbrcloud.finguard:finguard-ai`, Java 21, LangChain4j, TestNG, Jackson, SLF4J. |
| `mvnw`, `mvnw.cmd` | Maven Wrapper for reproducible builds without a global Maven install. |

### `src/main/java/com/rbrcloud/finguard/`

| Path | Purpose |
|------|--------|
| **evaluator/** | Components that judge whether AI responses are grounded in the knowledge base. |
| **evaluator/GroundednessEvaluator.java** | Uses `KnowledgeBase` to get product truth; validates that loan amounts and APRs mentioned in a response are within allowed ranges (e.g. Personal Loan max amount, APR min/max). |
| **evaluator/KnowledgeBase.java** | Loads `/rbr-products.json` from classpath; exposes `getProductData(String productName)` returning product JSON for evaluators. |
| **sut/** | System under test—simulates the AI agent. |
| **sut/RbrLoanAgent.java** | Mock agent: maps queries (e.g. "personal loan", "loan amount", "angry customer") to fixed responses; includes intentional wrong answers (e.g. $100k max) for hallucination tests. |

### `src/main/resources/`

| Path | Purpose |
|------|--------|
| `application.yml` | Application configuration (currently empty). |
| `rbr-products.json` | Canonical product data: Personal Loan (APR, amounts, terms), RBR Card, Rewards Checking Plus. Used by `KnowledgeBase` and evaluators. |

### `src/test/java/com/rbrcloud/finguard/suites/`

| Path | Purpose |
|------|--------|
| **HallucinationTests.java** | TestNG suite: calls `RbrLoanAgent` with queries and asserts `GroundednessEvaluator` finds responses grounded (e.g. max loan amount ≤ KB max, APR within range). Includes a flaky scenario (personal loan + loan amount) to simulate intermittent hallucination. |
| **ComplianceTests.java** | TestNG suite: drives `RbrLoanAgent` responses through `SemanticsComplianceEvaluator` (LLM-based) to ensure responses meet semantic/compliance requirements (expects a `PASS` result). |

### `src/test/resources/`

Reserved for test-only configuration or data (e.g. alternate product JSON, test YAML). Currently no files.

### `target/`

Generated by Maven: compiled classes, JARs, test results. Typically not committed.

### `.idea/`

IntelliJ IDEA project and workspace settings. Optional; often excluded from version control via `.gitignore`.

## Knowledge base (`rbr-products.json`)

Defines the source of truth for product facts:

- **Personal Loan:** `min_apr`, `max_apr`, `min_amount`, `max_amount`, `origination_fee_range`, `terms`
- **RBR Card:** `apr_range`, `rewards`, `annual_fee`, `credit_line_max`
- **Rewards Checking Plus:** `cashback`, `apy`, `monthly_fee`

Evaluators (e.g. `GroundednessEvaluator`) use this to reject responses that state amounts or APRs outside these bounds.

## Build and test

```bash
# Build
./mvnw clean compile

# Run tests
./mvnw test
```

Tests live in `HallucinationTests` and `ComplianceTests`: they drive `RbrLoanAgent` and assert groundedness via `GroundednessEvaluator` and semantic/compliance behavior via `SemanticsComplianceEvaluator`.

---

## Summary

- **Evaluators** (`evaluator/`) check that AI answers match the **knowledge base** (`rbr-products.json`).
- **SUT** (`sut/`) is a mock loan agent used to demonstrate and test hallucination detection.
- **Suites** (`suites/`) contain TestNG tests that tie SUT and evaluators together for regression and validation.
